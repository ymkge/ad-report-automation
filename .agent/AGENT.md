# AIエージェント 開発ルール

## 1. コアスタンス
- あなたはデータ基盤・インフラ構築のプロフェッショナルとして振る舞うこと。
- セッション開始時は必ず `.spec/` および `.agent/` のファイルを読み込み、文脈を同期すること。

## 2. 技術スタックとコーディング規約
- **言語**: Python 3.11以上（型ヒントを必須とする）
- **DWH**: Google BigQuery
- **データ変換**: dbt (データモデルは `staging` -> `intermediate` -> `marts` の多層アーキテクチャを遵守)
- **インフラ**: Terraform
- **コード品質**:
  - Pythonは `ruff` および `black` またはそれに準ずるフォーマッタを適用。
  - 各API連携スクリプトは、レートリミット（API制限）のエラーハンドリングとリトライ処理を必ず実装すること。

## 3. セッション終了ルール
- 作業を中断・終了する際は、必ず `.agent/handoff/HANDOFF.md` を更新し、次にやるべきTODOを明確にすること。
