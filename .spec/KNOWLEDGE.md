# 開発ナレッジ・認証情報取得手順

## 1. Google Ads API 認証情報取得フロー
非エンジニアの運用を想定し、以下の情報をクライアントから受領、または共同で設定します。

### 受領・設定が必要な項目
- **MCCアカウントID**: 顧客アカウントを束ねる管理用ID（XXX-XXX-XXXX）。
- **開発者トークン (Developer Token)**: Google Ads APIを利用するためのキー。
  - 手順: MCCアカウントの「ツールと設定」＞「APIセンター」から申請。最初は「テストアクセス」でOK。
- **Google Cloud プロジェクト**:
  - `Google Ads API` を有効化。
  - **OAuth 2.0 クライアントID / シークレット**: GASからアクセスするために必要。
  - **リフレッシュトークン**: 永続的なアクセスに必要（GAS側で一度認証して取得）。

## 2. Meta Ads API 認証情報取得フロー
Metaは「システムユーザー」を利用した「無期限アクセストークン」の取得を推奨します。

### 受領・設定が必要な項目
- **広告アカウントID**: `act_xxxxxxxxxxxxxxx` の形式。
- **Metaアプリ**:
  - 「ビジネス」タイプで作成。
  - `Marketing API` を追加。
- **システムユーザー (System User)**:
  - ビジネスマネージャーの「設定」＞「ユーザー」＞「システムユーザー」から作成。
  - 広告アカウントへの「広告アカウントの管理（full-control）」権限を付与。
- **アクセストークン**:
  - システムユーザーの「トークンを生成」から発行。
  - 有効期限を「無期限」に設定。
  - スコープ: `ads_read`, `read_insights` を含める。

## 3. セキュリティ上の注意点
- 取得したトークンやシークレットは、絶対にGitHub等のソースコード管理ツールに直接コミットしない。
- スプレッドシートの「設定シート」に保存し、シート自体の共有範囲を適切に制限することで管理する。
